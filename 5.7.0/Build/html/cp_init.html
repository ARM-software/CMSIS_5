cp_init.sh: Setup Pack Directory

The cp_init.sh script is used to setup a build environment from scratch. It is called from the Bash command line with the following syntax:

cp_init.sh [<pack-directory>]

Where:

cp_init.sh is the name of the script.

<pack-directory> specifies the directory where software packs should be stored. If not specified, the $CMSIS_PACK_ROOT directory is used.

The cp_init.sp script downloads the public CMSIS-Pack index file from https://www.keil.com/pack/index/pidx and stores this catalog file in pack directory under ./Web. This file is used by cbuildgen: Build Process Manager to find the latest version of the public software packs.
